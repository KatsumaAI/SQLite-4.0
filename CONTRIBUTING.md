# Contributing to SQLite-4.0

Thank you for your interest in contributing to SQLite-4.0! This document provides guidelines and instructions for contributing.

## Ways to Contribute

- **Bug Reports** - Report bugs and issues
- **Feature Requests** - Suggest new features
- **Code Contributions** - Submit pull requests
- **Documentation** - Improve docs and examples
- **Testing** - Help test and verify features
- **Security** - Responsible disclosure of vulnerabilities

## Getting Started

### Prerequisites

- Node.js 14.0.0 or higher
- Git
- npm or yarn

### Development Setup

1. Fork the repository on GitHub
2. Clone your fork locally:

```bash
git clone https://github.com/YOUR-USERNAME/SQLite-4.0.git
cd SQLite-4.0
```

3. Install dependencies:

```bash
npm install
```

4. Create a feature branch:

```bash
git checkout -b feature/amazing-feature
```

5. Make your changes
6. Run tests:

```bash
npm test
```

7. Commit your changes:

```bash
git commit -m 'Add amazing feature'
```

8. Push to GitHub:

```bash
git push origin feature/amazing-feature
```

9. Create a Pull Request

## Code Style

### JavaScript

- Use ES6+ features
- Follow consistent indentation (2 spaces)
- Use semicolons
- Use const for constants, let for variables
- Use template literals for string interpolation
- Use arrow functions when appropriate

### Comments

- Use JSDoc comments for functions
- Document complex logic
- Explain "why" not just "what"

### Testing

- Write tests for new features
- Ensure all tests pass before submitting
- Include edge cases in tests

## Feature Requests

Before submitting a feature request:

1. Check if the feature already exists
2. Check if it's planned for a future version
3. Consider if it aligns with the project goals

Provide:
- Clear description of the feature
- Use cases
- Example code (if applicable)
- Related issues or PRs

## Bug Reports

Include:
- Clear description of the bug
- Steps to reproduce
- Expected behavior
- Actual behavior
- Screenshots (if applicable)
- Environment details (OS, Node version)

## Security

If you discover a security vulnerability, please:

1. DO NOT open a public issue
2. Email: security@katsuma.ai
3. We will acknowledge receipt within 24 hours
4. We will work with you to responsibly disclose
