Introduction

SQLite-4.0 is a secure embedded SQL database for modern Node.js applications. It provides enterprise-grade security features including AES-256-GCM encryption, role-based access control, and comprehensive observability tools.

Key Features

Zero dependencies, full encryption, chainable APIs, TypeScript support, and enterprise features like replication and monitoring.

Installation

Install SQLite-4.0 via npm or clone from GitHub:

# Via npm
npm install sqlite4

# From GitHub
git clone https://github.com/KatsumaAI/SQLite-4.0.git
cd SQLite-4.0 && npm install

Quick Start

Get started with a basic database instance:

const SQLite4 = require('sqlite4');

// Create database with encryption
const db = new SQLite4({
  dbPath: './data/app.db',
  key: 'your-secret-key'
});

// Execute queries
db.execute(`CREATE TABLE users (id INTEGER PRIMARY KEY, name TEXT)`);
db.execute(`INSERT INTO users (name) VALUES ('Alice')`);
const result = db.execute(`SELECT * FROM users`);
console.log(result.rows);

Encryption

SQLite-4.0 uses AES-256-GCM encryption for data at rest. All data is encrypted when the database key is provided.

Encryption Key

The encryption key should be 32 characters or longer for optimal security:

const db = new SQLite4({
  dbPath: './data/app.db',
  key: process.env.DB_KEY
});
Security Note

Never commit your encryption key to version control. Use environment variables or a secure secrets manager.

Authentication

SQLite-4.0 supports user authentication with role-based access control.

Adding Users

// Add users with roles
db.addUser('admin', 'securePassword123', 'admin');
db.addUser('readonly', 'readonlyPass', 'readonly');

Authenticating

// Authenticate user
const user = db.authenticate('admin', 'securePassword123');
if (user) {
  console.log(`Authenticated as ${user.role}`);
}

Permissions

SQLite-4.0 uses a role-based permission system:

Role Permissions
admin SELECT, INSERT, UPDATE, DELETE, CREATE, DROP, ALTER, GRANT, REVOKE
operator SELECT, INSERT, UPDATE, DELETE
readonly SELECT only
viewer SELECT only

Query Builder

Use the chainable query builder for type-safe queries:

// Chainable query builder
const users = await db
  .select('name', 'email')
  .from('users')
  .where('status', 'active')
  .orderBy('created_at', 'DESC')
  .limit(10)
  .all();

Schema Builder

Create tables programmatically with the schema builder:

// Programmatic schema creation
db.schema.create('posts', (table) => {
  table.increments('id');
  table.string('title', 255);
  table.text('content');
  table.integer('user_id').references('users', 'id');
  table.timestamp('created_at');
});

Migrations

Manage database schema changes with version-controlled migrations:

// Create a migration file
db.migration.create('add_users_table', 'CREATE TABLE users...', 'DROP TABLE users');

// Run migrations
await db.migration.migrate();

// Rollback last batch
await db.migration.rollback();

Replication

Set up master-slave replication for high availability:

const replication = new SQLite4.Replication({
  master: './data/master.db',
  slaves: ['./data/slave1.db', './data/slave2.db'],
  syncInterval: 5000
});

replication.start();

Backups

Create encrypted, compressed backups:

// Create backup
await db.backup('./backups/app_backup', {
  compress: true,
  encrypt: true,
  password: process.env.BACKUP_PASSWORD
});

Docker

Run SQLite-4.0 in Docker:

# Basic container
docker compose up -d

# With TLS
docker compose -f docker-compose.tls.yml up -d

# High availability
docker compose -f docker-compose.ha.yml up -d

Monitoring

SQLite-4.0 includes Prometheus-compatible metrics:

const metrics = new SQLite4.Metrics({
  prefix: 'sqlite4_',
  exportDir: './metrics'
});

// In Express
app.get('/metrics', metrics.httpHandler());
