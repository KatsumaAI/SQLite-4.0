API Reference

Complete API documentation for SQLite-4.0 including all modules, methods, and TypeScript definitions.

SQLite4 (Main Class)

The main database class for creating and managing SQLite-4.0 databases.

new SQLite4(options)

Creates a new SQLite-4.0 database instance.

dbPath string Path to database file, or ':memory:' for in-memory database
key string Encryption key for AES-256-GCM encryption
readonly boolean Open database in read-only mode (default: false)
timeout number Lock timeout in milliseconds (default: 5000)
const db = new SQLite4({
  dbPath: './data/app.db',
  key: process.env.DB_KEY
});
execute(sql, params?)

Executes a SQL statement and returns the result.

Returns: { success, rows, rowCount, columns, error, lastInsertId, changes }
query(sql, params?)

Executes a SELECT query and returns rows.

Returns: { success, rows, rowCount, columns }
all(sql, params?)

Executes query and returns all rows as an array.

Returns: any[]
get(sql, params?)

Executes query and returns the first row.

Returns: any | undefined
run(sql, params?)

Executes INSERT/UPDATE/DELETE and returns metadata.

Returns: { lastInsertId, changes }
prepare(sql)

Creates a prepared statement for reuse.

Returns: Statement
begin() / commit() / rollback()

Transaction control methods.

Returns: boolean
addUser(username, password, role)

Adds a new user with specified role.

authenticate(username, password)

Authenticates a user and returns user object.

Returns: { username, role } | null
backup(destination, options)

Creates an encrypted, compressed backup.

close()

Closes the database connection.

QueryBuilder

Chainable API for building type-safe SQL queries.

const users = await db
  .select('id', 'name', 'email')
  .from('users')
  .where('status', 'active')
  .where('role', 'admin')
  .orderBy('created_at', 'DESC')
  .limit(10)
  .all();

SchemaBuilder

Programmatic table creation with chainable API.

db.schema.create('posts', (table) => {
  table.increments('id');
  table.string('title', 255);
  table.text('content');
  table.integer('user_id');
  table.foreignKey('user_id', { table: 'users', column: 'id' });
  table.timestamp('created_at');
});

Migration

Version-controlled schema migrations.

// Create migration
db.migration.create('create_users',
  'CREATE TABLE users...',
  'DROP TABLE users'
);

// Run migrations
await db.migration.migrate();

// Rollback
await db.migration.rollback();

Methods

create(name, upSQL, downSQL)

Creates a new migration file.

migrate(options)

Runs pending migrations (direction: 'up' | 'down').

rollback()

Rolls back the last batch of migrations.

fresh()

Drops all tables and re-runs all migrations.

getStatus()

Returns migration status.

Returns: { executed, pending, currentBatch, lastExecuted }

ConnectionPool

Efficient connection management with pooling.

const pool = new SQLite4.ConnectionPool({
  min: 2,
  max: 10,
  acquireTimeout: 30000,
  idleTimeout: 60000
});

const conn = await pool.acquire();
// Use connection...
pool.release(conn);

Metrics

Prometheus-compatible metrics collection.

const metrics = new SQLite4.Metrics({
  prefix: 'sqlite4_',
  exportDir: './metrics'
});

// Track query
const tracker = metrics.trackQuery();
const t = tracker.start('SELECT');
// Execute query...
t.done();

// Export metrics
app.get('/metrics', metrics.httpHandler());

DatabaseEvents

Event system for hooks and logging.

const events = new SQLite4.DatabaseEvents(db);

events.on('query', (data) => {
  console.log(data.sql, data.duration + 'ms');
});

events.on('authSuccess', (data) => {
  console.log('User logged in:', data.user);
});
